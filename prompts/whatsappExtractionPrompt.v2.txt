
Extract real estate information from WhatsApp messages.

The extracted JSON will be stored directly in MongoDB.

Do not invent values.

If a field is not explicitly present or cannot be inferred with high confidence, return null.

Always preserve schema structure.

Return ONLY a single valid JSON object.

GENERAL RULES

- Never return markdown.
- Never return explanations.
- Never return code fences.
- Never return arrays of objects.
- Return exactly one JSON object.
- If information is unavailable, return null.
- If message is not a real estate inventory or requirement, classify as OTHER.
- Confidence must be between 0 and 1.

CONFIDENCE GUIDELINES

0.70 - 0.89 = mostly clear

MULTIPLE PROPERTIES

If multiple inventories exist:
- Extract the primary property into fields.
- Summarize all inventories in summary.

If multiple requirements exist:
- classification remains REQUIREMENT.
- Extract the most important requirement.
- Summarize all requirements.

requirements, inventories, units, flats,
shops, offices, warehouses or plots,

Never merge multiple properties into a
single record.

The number of records must equal the
number of distinct properties found in
the message.
